ANÁLISIS TEÓRICO DEL DESAFÍO DE TETRIS

El corazón del desafío es representar y manipular un tablero de juego y las piezas (tetriminos) de una manera que no es la intuitiva (matrices 2D), sino usando operaciones a nivel de bits para maximizar la eficiencia. Esto implica cambiar la forma en la que se entienden los datos y cómo se van a manejar dentro del programa.

1. Representación de Datos con Bits: El Tablero
   Dado que el ancho del tablero debe ser múltiplo de 8, se aprovecha que 8 bits forman un byte. Esto permite representar cada fila del tablero como uno o varios números enteros sin signo.

La idea es trabajar el tablero usando memoria dinámica, por medio de un arreglo de punteros (unsigned char**), donde cada fila contiene varios bytes dependiendo del ancho.

Cada bit dentro de esos bytes representa una celda:

* 1 → ocupado
* 0 → vacío

Para la visualización, se recorrerán los bytes y luego cada uno de sus bits para mostrar el estado del tablero en pantalla.

Creación del tablero: Para un tablero de ancho W y alto H, se plantea un arreglo dinámico de H filas, donde cada fila contiene W/8 bytes.

Ajuste del tamaño: Se usa memoria dinámica para permitir que el usuario defina el tamaño del tablero, con un mínimo de 8 y con la condición de que el ancho sea múltiplo de 8.

2. Representación de Datos con Bits: Las Piezas (Tetraminós)
   Cada pieza se representa como una máscara de bits. Cada tetrimino se maneja como un arreglo de hasta 4 filas, donde cada fila es un número cuyos bits indican la forma.

El uso de literales binarios (0b) permite que la forma de cada pieza sea más clara al momento de definirla.

Además, cada pieza tiene asociado un ancho y un alto que definen sus dimensiones.

Las piezas se plantean como elementos que pueden generarse y manipularse para luego ser utilizadas dentro del tablero.

3. Manipulación: Operadores a Nivel de Bits (Bitwise)
   Las operaciones a nivel de bits son la base para toda la lógica del programa:

Desplazamiento (<<, >>): Se utilizarán para mover bits dentro de una fila, representando movimientos horizontales.

AND bitwise (&): Se utilizará para verificar posibles colisiones entre piezas y el tablero.

OR bitwise (|): Se usará para activar bits, por ejemplo al colocar una pieza dentro del tablero.

XOR bitwise (^): Puede emplearse en algunos casos específicos.

NOT bitwise (~): Se utilizará para construir máscaras de control o límites.

4. Movimiento de piezas
   El movimiento de las piezas se plantea como la modificación de la posición de los bits.

Mover a la izquierda o derecha implica desplazar bits dentro de la fila, mientras que mover hacia abajo implica cambiar la fila en la que se encuentra la pieza.

Estas acciones se estructuran mediante funciones que controlan cada tipo de movimiento.

5. Rotación de Piezas
   La rotación se plantea como una reorganización de los bits que forman la pieza.

Para esto, se considera trabajar con una representación cuadrada de la pieza, donde cada bit se reubica en una nueva posición mediante el uso de máscaras y desplazamientos.

Es una de las operaciones más complejas dentro del sistema.

6. Gestión de Memoria Dinámica y Punteros

Asignación Dinámica: Se utilizará memoria dinámica (new) para crear el tablero, ya que su tamaño se define en tiempo de ejecución.

Punteros para eficiencia: Se emplean para manejar las filas del tablero y facilitar operaciones sobre ellas.

Limpieza de filas: Se plantea detectar filas completas verificando que todos los bits estén en 1, para luego reorganizar las filas del tablero.

Gestión de memoria: Se considera la liberación de memoria como parte del manejo adecuado del sistema.

CONCLUSIÓN
Este análisis plantea el uso de bits como base para representar tanto el tablero como las piezas, junto con las operaciones necesarias para manipularlos. Se define la estructura general del sistema, incluyendo la creación del tablero, la representación de las piezas y las operaciones fundamentales que permiten el funcionamiento del juego..
